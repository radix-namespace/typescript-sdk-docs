Radix Namespace SDK

TypeScript SDK for registering domains, managing records, and building on the Radix Namespace protocol. This reference covers every public method, its parameters, response shapes, and usage examples.

No methods match your search.

Getting Started

Install the SDK and create your first instance.

Installation
Terminal 
npm install @radixnamespace/typescript-sdk

Peer dependencies: @radixdlt/radix-dapp-toolkit and @radixdlt/babylon-gateway-api-sdk must also be installed in your project.

Initialization

Create an SDK instance with your Radix Dapp Toolkit and target network. The SDK auto-discovers on-ledger component addresses on first use.

TypeScript
import NamespaceSDK from '@radixnamespace/typescript-sdk';
import { RadixDappToolkit } from '@radixdlt/radix-dapp-toolkit';

const rdt = RadixDappToolkit({ /* your config */ });

const namespace = new NamespaceSDK({
  rdt,
  network: 'stokenet'  // or 'mainnet'
});
Response Format

All SDK methods return a discriminated union. Query methods return SdkResponseT<T> and transaction methods return SdkTransactionResponseT<T>. Check for the presence of data/feedback or errors to determine success.

Query Response Shape
// Success
{ data: T }

// Error
{ errors: [{ code: "ERROR_CODE", error: "Human message", verbose: "Details" | null }] }
Transaction Response Shape
// Success
{ feedback: { messages: [{ code: "TX_SUCCESS", details: "..." | null }] } }

// Error
{ errors: [{ code: "ERROR_CODE", error: "Human message", verbose: "Details" | null }] }
Tip: Transaction methods accept an optional callbacks parameter with hooks: onInit, onAppApproved, onIntentReceipt, onSuccess, onFail.
